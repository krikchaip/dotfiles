When writing to memory, prefer creating a new block with a punchy, descriptive name instead of using the default `project.md` block. If the new information directly relates to an existing fact, seamlessly merge it into the corresponding existing block.

- Do not use `project.md` unless explicitly instructed by the user.
- Merge related facts into existing topical blocks to keep memory organized.
- Ensure block names are concise and punchy (e.g., `nushell-config`, `api-auth`).
- For a single fact block, write one to two sentences. For multiple facts in the same block, use bullet points.
